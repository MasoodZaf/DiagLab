# White-Label System

## Branding layers

### 1. Core invariant product behavior

- shared workflow structure
- shared navigation model
- shared domain contracts
- shared permission model

### 2. Theme token layer

- typography pair
- primary and neutral palette
- surface elevation and border style
- radius scale
- density scale
- chart palette

### 3. Tenant content/config layer

- brand name and copy tone
- domains and locale defaults
- report header and footer
- notification templates
- feature flags
- hero imagery direction

## v1 guardrails

- No tenant-specific page builders
- No tenant-specific workflow logic outside feature flags and policy config
- Product-owned layout system across operations views
- Highest branding priority on patient surfaces, auth, reports, and notifications

## Required tenant assets

- horizontal logo
- square logo / app icon
- favicon
- approved brand colors
- typography pairing
- report signature block
- notification sender names
